# News_TEAM5

## 컨벤션
### Code Convention

- 클래스: `PascalCase`
- 변수명: `snake_case`
- 함수명: `snake_case`
- 상수: `SNAKE_CASE`

### folder and files Convention
가능하면 영문 소문자로만 구성한다.  
가능하면 짧게 구성한다(축약어 사용).  
특수문자와 빈 공백sᴘᴀᴄᴇ은 사용하지 않는다.  
단어와 단어의 구분은 ‘_’(ᴜɴᴅᴇʀʙᴀʀ) 로 구성한다.  

### Git Convention

기본 브랜치: `main`

<aside>
💡 `COMMIT_TYPE: COMMIT_SUMMARY`

</aside>

- **COMMIT_TYPE**
    - feat : 새로운 기능 추가
    - fix : 버그 수정
    - docs : 문서 추가 및 수정
    - style : 코드 포맷팅, 세미콜론 누락, 오타 수정 등
    - test : 테스트코드
    - refactor : 코드 리팩토링
    - chore : 빌드 업무 수정, 패키지 매니저 수정
- **COMMIT_SUMMARY**
    - 영어로 작성
    - 마침표를 붙이지 않음
    - 50자를 넘기지 않음

- **BRANCH_NAME**  
crawler, preprocess, modeling, api  
  
### etc
